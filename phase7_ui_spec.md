## 整体路由

```
/            → 首页（标题画面）
/create      → 捏人页（4步向导 → 写入 CharacterState → 创建存档 → 开始游戏）
```

### 2.1 🔌 API 配置

**用途**：管理 AI 模型连接端点。支持 OpenAI 兼容 API。

**当前实现**：
- 端点列表：每项显示名称 / 地址 / 模型 / [测试连接] [删除]
- 添加端点表单：名称 / 地址 / 模型 / API Key
- 连接测试：点击测试 → toast 通知结果

**待补充**：
- 默认端点标记（当前使用哪个端点）
- 端点编辑（目前只有添加/删除）
- 端点优先级排序
- 模型列表自动获取（/v1/models）

### 2.2 🤖 Agent 配置

**用途**：为每个 Agent 单独指定模型、温度、最大 Token 等参数。

**当前实现**：
- 卡片列表展示 11 个 Agent：`memory_recall`, `plot_pre_check`, `story`, `vars_update`, `char_update`, `memory_summary`, `plot_post_check`, `plot_outline`, `craft_gen`, `char_gen`, `item_gen`
- 每个 Agent 显示：名称 / 当前模型 / 温度 / max_tokens / 启用开关

**待补充**：
- 点击 Agent 卡片展开详细配置（模型选择 / 温度滑块 / maxTokens 输入 / 超时 / 重试次数）
- 全局默认值设置（所有 Agent 的默认模型/温度/maxTokens）
- Agent 描述文字（这个 Agent 做什么的）
- 预设方案保存/加载（如"高速模式""质量优先"）

---

### 2.3 🔗 Agent 管线

**用途**：编排 Agent 执行顺序。同 Stage 内 Agent 并行执行。

**当前实现**：
- Stage 0-5 卡片，每张显示该 Stage 的 Agent 名称和执行说明
- 每 Stage 有启用开关

**管线结构**：
```
Stage 0: memory_recall + plot_pre_check（并行）
Stage 1: story（正文生成，最关键的 Agent）
Stage 2: vars_update（变量更新）
Stage 3: char_update（角色更新）
Stage 4: memory_summary（记忆总结）
Stage 5: plot_post_check（剧情修正）
```

**待补充**：
- 管线 DAG 可视化图（节点+连线）
- 拖拽调整 Stage 顺序
- 拖拽调整 Agent 归属 Stage
- 每 Stage 的超时设置
- 管线预设（如"快速模式"跳过后 Stage）

### 2.4 📖 剧情设置

**用途**：控制剧情生成模式和主线参数。

**当前实现**：
- 剧情模式选择：完全关闭 / 仅支线 / 主线
- 主线配置：
  - 持续年份（1-50）
  - 事件难度层级（1-7，对应生命层级）
  - 是否引入世界书外 NPC
  - 剧情偏向多选：战斗 / 解密 / 社交 / 恋爱
  - 自定义偏好输入框
- 支线配置：
  - 每年自动生成
  - 专注区域

**待补充**：
- 主线大纲预览（当前生成的大纲内容）
- 世界线变动传播深度
- 剧情事件完成/失败时自动生成记忆的开关
- "生成大纲"按钮 → 调用 plot_outline Agent

### 2.5 🧠 记忆 & 缓存

**用途**：控制 Embedding 召回、记忆压缩和缓存策略。

**当前实现**（表单字段，无后端联动）：
- Embedding 端点选择
- Embedding 模型名
- 向量维度
- 每轮最大召回记忆数（5-50，默认 20）
- 压缩阈值（50-500 轮，默认 100）
- 每存档最大快照数（0-50，默认 30）
- 缓存策略选择：激进 / 平衡 / 保守

**待补充**：
- Embedding 端点测试按钮
- 当前记忆总数 / 平均轮次显示
- 手动触发压缩按钮
- 缓存命中率统计面板
- maxMemoriesRecall / compressionThreshold 实际写入 AppSettings
