# agent流程测试

本文件要求测试我们目前的agent在实例下运行的完备性，先从基础的准备流程开始
1. 从"游戏实例_对话.json" 找三到四个对话，只提取正文（即不包含think或者var标签），其中要求：包含一个和制造物品相关的正文，一个包含新角色的正文，一个日常对话（仅正常的变量更新的正文），然后写入一个markdown文件，放在这个文件的相同文件夹，我来检查。

2. 有了对话数据以后，先写预期效果，在新文件里写上：每个ai的预计上下文是什么，预计输出可能是什么样的，输出符不符合我们目前的架构/能不能被目前架构读取，可以分出数个agent，除了记忆召回ai，剧情相关的ai，别的ai都要测一下，总结一下就是：变量更新，角色更新，制作效果生成，角色生成，物品生成，每个ai都至少要对应一条正文对话，然后写入到文件里的格式大致为：
每个ai的职责
上下文内容（参考预设，世界书，变量和输入）
测试用正文
可能输出

然后写到新文件里
